# 下半年计算机顶会论文搜集 Agent 需求文档

## 1. 项目概述

### 1.1 项目目标

开发一个自动化 Agent，用于搜集下半年（7 月–12 月）召开的计算机领域顶级会议信息，并从 **arXiv** 中检索与各会议对应的预印本论文，筛选出作者所属单位为中国大陆及港澳地区的文章，进一步补全作者信息（中文名、所属单位），最终生成结构化 Markdown 报告并部署至 GitHub 实现自动化更新。

### 1.2 与现有项目的区别

| 维度 | 现有项目 | 本需求 |
|------|----------|--------|
| 数据源 | 会议官网、科技新闻、学校官网、微信公众号 | **arXiv** 为主 |
| 会议信息 | 时间、地点、CCF 评级、领域、投稿截止等 | **召开时间、召开地点** |
| 作者筛选 | 清华 / 北大高亮 | **国内（含港澳）所有作者** |
| 作者信息 | 英文名 + 机构 | **中文名 + 所属单位** |
| 输出 | 按会议分章节 + 清北统计 | 按会议分章节 + 国内作者汇总 |

### 1.3 适用范围

- 目标年份：当前年度下半年（默认 2026 年 7–12 月）
- 目标会议：CCF A 类计算机顶会（下半年召开）
- 作者地域：中国大陆、香港特别行政区、澳门特别行政区

## 2. 功能需求

### 2.1 会议信息采集

- **采集范围**：下半年（7 月–12 月）召开的 CCF A 类计算机顶会
- **采集字段**：
  - 会议全称及缩写
  - 召开时间（起止日期）
  - 召开地点（城市、国家）
  - 官方网站（可选）
- **数据来源**：会议官网、WikiCFP、CCF 推荐目录
- **维护方式**：配置文件维护会议清单，支持手动增删

### 2.2 arXiv 论文检索

- **检索策略**：
  - 按会议主题关键词检索 arXiv（如 `cs.CL` 对应 ACL/EMNLP，`cs.LG` 对应 ICML/NeurIPS）
  - 按会议缩写 + 年份检索（如 `ACL 2026`、`NeurIPS 2026`）
  - 按时间窗口筛选（会议召开当年及前一年的预印本）
- **采集字段**：
  - 论文标题
  - arXiv 编号（如 `2607.12345`）
  - 全部作者列表（英文名）
  - 作者所属机构（如有，arXiv 论文通常需通过其他途径补全）
  - 论文摘要
  - 论文链接（arXiv abs / pdf 链接）
  - arXiv 主题分类
  - 提交时间
- **数据来源**：arXiv API（`http://export.arxiv.org/api/query`）

### 2.3 国内作者筛选

- **地域范围**：中国大陆、香港、澳门
- **识别方式**：
  - 通过作者所属机构关键词匹配（如 `China`、`Tsinghua`、`Peking`、`Hong Kong`、`Macau` 等）
  - 通过机构归一化别名表（如 `THU → 清华大学`、`PKU → 北京大学`）
  - arXiv 本身不提供机构信息时，通过 Semantic Scholar / OpenAlex / Crossref 补全
- **筛选字段**：
  - 论文标题
  - 国内作者列表（标注其机构）
  - 论文链接

### 2.4 作者信息补全

- **补全目标**：
  - 作者中文名（如有，如 `Jianlin` → `林剑林`）
  - 作者所属单位（中文规范名，如 `清华大学计算机科学与技术系`）
- **补全来源**：
  - OpenAlex API（`last_known_institutions`、作者 `display_name`）
  - Semantic Scholar API（`authors.affiliations`）
  - 高校官网 / 个人主页（可选，需人工辅助）
  - 中文名推断：拼音反查 / 已知华人学者库
- **输出格式**：
  - 英文名 / 中文名 / 所属单位（中文）

### 2.5 报告生成

- **格式**：结构化 Markdown
- **组织方式**：按会议召开时间顺序排列，每个会议独立章节
- **章节内容**：
  - 会议基本信息（时间、地点、官网）
  - 国内作者论文列表（含作者中文名、所属单位）
  - 该会议国内论文统计
- **总览统计**：
  - 会议总数
  - arXiv 检索论文总数
  - 国内作者论文总数
  - 各机构中稿统计（按单位聚合）

### 2.6 自动化更新

- 支持定时执行（GitHub Actions cron）
- 增量更新：仅检索新增 / 变动的论文
- 自动提交并推送至 GitHub 仓库

## 4. 技术架构

### 4.1 核心模块

1. **配置管理模块**：维护会议清单、arXiv 分类映射、国内机构关键词
2. **arXiv 采集模块**：调用 arXiv API 检索论文，解析 Atom XML
3. **机构补全模块**：通过 OpenAlex / Semantic Scholar 补全作者机构
4. **国内筛选模块**：机构归一化、国内（含港澳）识别、中文名推断
5. **报告生成模块**：Jinja2 Markdown 模板渲染
6. **Git 同步模块**：自动 commit + push 至 GitHub

### 4.2 技术栈

- **开发环境**：Cursor IDE + Python 3.11
- **核心库**：`requests`、`feedparser`（arXiv Atom 解析）、`PyYAML`、`Jinja2`、`GitPython`
- **数据源**：
  - arXiv API：`http://export.arxiv.org/api/query`
  - OpenAlex API：`https://api.openalex.org`
  - Semantic Scholar API：`https://api.semanticscholar.org/graph/v1`
- **部署**：GitHub Actions 定时运行

### 4.3 项目结构

```
conference-tracker/
├── config/
│   ├── conferences.yaml      # 会议清单 + arXiv 分类映射
│   ├── institutions.yaml      # 国内机构关键词 + 别名表
│   └── settings.yaml          # 运行参数
├── src/
│   ├── collector/

│   │   ├── affiliation_enricher.py # 机构补全（OpenAlex/S2）
│   │   └── base.py

│   │   └── deduplicator.py
│   ├── generator/
│   │   └── report_generator.py
│   └── sync/
│       └── git_sync.py
├── templates/
```

## 6. 非功能需求

- **可扩展性**：支持新增会议、新增 arXiv 分类、新增机构关键词
- **可维护性**：模块化设计，配置与代码分离
- **容错性**：单数据源失败不影响整体运行；arXiv 限流时自动退避重试
- **合规性**：遵守 arXiv API 使用条款（请求间隔 ≥ 3 秒，单次 ≤ 300 条）
- **性能**：单次全量采集应在 30 分钟内完成

## 7. GitHub 部署方案

### 7.1 仓库结构

```
your-github-repo/
├── .github/workflows/update-report.yml
├── reports/2026-H2-conferences.md
├── src/
├── config/
├── main.py
└── requirements.txt
```

### 7.2 GitHub Actions 工作流

```yaml
name: Update Conference Report

on:
  schedule:
    - cron: '0 0 * * 1'   # 每周一 UTC 00:00 执行
  workflow_dispatch:       # 支持手动触发

permissions:
  contents: write

jobs:
  update:
    runs-on: ubuntu-latest
    timeout-minutes: 60
    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: 'pip'

      - name: Install dependencies
        run: pip install -r requirements.txt

      - name: Run collector
        run: python main.py --full --push
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GIT_USER_NAME: github-actions[bot]
          GIT_USER_EMAIL: github-actions[bot]@users.noreply.github.com
```

### 7.3 部署步骤

1. 在 GitHub 创建空仓库（不勾选 README）
2. 本地执行：
   ```powershell
   git remote add origin https://github.com/<用户名>/<仓库名>.git
   git branch -M main
   git push -u origin main
   ```
3. 仓库 **Settings → Actions → General**，将 **Workflow permissions** 设为 **Read and write permissions**
4. **Actions** 页面手动触发一次 **Update Conference Report** 验证

## 8. 命令行接口

```bash
# 增量更新全部会议
python main.py

# 全量采集
python main.py --full

# 仅采集指定会议
python main.py --conference icml-2026 --full

# 采集并推送至 GitHub
python main.py --full --push

# 跳过机构补全（加快速度）
python main.py --full --no-enrich
```

| 参数 | 说明 |
|------|------|
| `--full` | 全量采集（默认增量） |
| `--conference ID` | 指定会议 ID |
| `--push` | 自动 git commit + push |
| `--force` | 强制重新采集（忽略缓存） |
| `--no-enrich` | 跳过 OpenAlex 机构补全 |
| `--no-report` | 跳过报告生成 |
